Best Practices

Learn smart

Use architecture docs, guidelines, PR's, designs, screenshots, test reports etc. in generating agent profiles.

Feedback-loop

Let the agent iterate itself: code → verify → improve. Build in automated feedback loops.

Confidence Thresholds

Build workflow rules around iteration and confidence levels. Know when to escalate.

RIGHT & WRONG Examples

Critical: Give agents examples of what's correct AND what's wrong. Context is everything.

  • Improve: Tell agents about your docs and guidelines - they'll use them!
  • Built-in improvement: Make profile refinement part of your workflow
  • Auto-verification: Tests, linters, type checkers = agent feedback
  • Confidence: Know when the agent should stop and ask for help
  • RIGHT/WRONG examples: The most underrated technique - show both!

Example: The Perfect Agent Task

Task: Rewrite Unit Tests — Old Framework -> New Framework

  • Focused scope — One framework swap, 1 trial project
  • Clear requirements — Compiles, tests pass
  • RIGHT examples — New Moq patterns to follow
  • WRONG examples — Old RhinoMocks patterns to replace
  • Auto-verify — It can Compile and Run tests = instant feedback loop
The agent could iterate independently: change code → run tests → fix failures → repeat
  • Real example from our codebase
  • Perfect task because scope is bounded and verification is built-in
  • RIGHT examples: Show the new Moq syntax you want
  • WRONG examples: Show the old RhinoMocks patterns to avoid
  • Tests provide automatic feedback - agent knows when it's done
