Feedback-loop

Confidence Thresholds

Build workflow rules around iteration and confidence levels. Know when to escalate.

RIGHT & WRONG Examples

Critical: Give agents examples of what's correct AND what's wrong. Context is everything.

  • Improve: Tell agents about your docs and guidelines - they'll use them!
  • Built-in improvement: Make profile refinement part of your workflow
  • Auto-verification: Tests, linters, type checkers = agent feedback
  • Confidence: Know when the agent should stop and ask for help
  • RIGHT/WRONG examples: The most underrated technique - show both!

MCP — Model Context Protocol

One standard way to expose context + tools to your AI agent

  • MCP = Model Context Protocol, introduced by Anthropic in late 2024
  • Problem it solves: every tool had its own integration — MCP standardises this
  • Left side: sources (docs, issues, databases/APIs)
  • Right side: AI agent gains the ability to read, query, and act on those sources
  • MCP is the bridge — one protocol, works across all supporting agents

MCP Servers — Real Examples

Atlassian

Read Jira issues and Confluence pages — give agents your full project context

MSSQL

Query databases directly — agents can inspect schemas and fetch live data

Playwright

Control a real browser — agents can navigate, fill forms, and test UIs

Hundreds of community-built servers at mcp.so
  • Atlassian: I use this to give agents context from Jira and Confluence without copy-pasting
  • MSSQL: Agent can query the DB directly — great for data migrations and reporting tasks
  • Playwright: The agent literally drives a browser — useful for E2E tests and UI automation
  • mcp.so is the community directory — most popular tools already have an MCP server

Simple to Configure

Add to your agent config:


{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
That's it. Github Copilot now has a real browser.
  • This is the entire Playwright MCP config — one command, one arg
  • npx downloads and runs the server on demand — nothing to install globally
  • For Github Copilot: add this to .claude/mcp.json or use "claude mcp add"
  • Other servers add env vars for credentials (API keys, connection strings) — still just JSON

Build Your Own MCP Server

Any existing API can become an MCP tool — just describe it:

"We have a REST API at /api/finance with endpoints for NPV, IRR, and amortization calculations. Create an MCP server that exposes these as tools so AI agents can call them directly."
Agents can now call your domain logic directly — no API wrapper knowledge needed
  • You don't need to understand the MCP spec to build a server — just describe your API
  • Github Copilot generates the server: handles tool definitions, request/response mapping, error handling
  • Financial calc example: now any agent can call NPV() or IRR() as a first-class tool
  • Same pattern works for internal APIs, legacy services, anything with HTTP endpoints
  • Once published, every team member's agent can use it instantly

Skills

Reusable know-how, deterministic helpers, and only the context it needs

  • Skills = slash commands that bundle instructions + domain knowledge + helper files
  • Three key ideas: domain knowledge (what things mean), deterministic helpers (SQL/scripts that always run the same way), progressive disclosure (only load relevant context)
  • Unlike a plain prompt, a skill can include SQL queries, markdown domain docs, and precise "when to use me" instructions
  • The agent doesn't need to figure out how to query your DB — the skill already has the right SQL

The window is finite — when it fills, compact into a summary and keep working

  • Context window = the "working memory" of the model for a conversation
  • Early in a conversation context is sharp; as it fills older content gets less attended to ("rotting")
  • Hard cutoff at 1M tokens — conversation simply stops if you hit it
  • Compact: summarise the conversation history, prepend the summary, continue fresh
  • Every turn the full context window is sent to the model: message history, instructions, MCP tools, and selected skills
  • No memory between calls — the model starts fresh each time with only what you send
  • Skills use progressive disclosure: only relevant skill details are injected, keeping the window focused and efficient

Context Health Meter

Be aware of where you are — optimize for quality, not capacity

0 – 20%
PRIME ZONE
Best focus, strongest reasoning, least clutter
20 – 60%
OK ZONE
Still workable, but watch for drift and unnecessary noise
60 – 95%
RESET ZONE
Start thinking about summarizing and clearing
95%+
DANGER ZONE
Auto-compaction may kick in; detail loss is likely and quality may already be worse
Not a target to fill — a resource to protect
  • Prime zone: fresh context = sharpest output. Start new conversations deliberately.
  • OK zone: normal working range, but be conscious of what you're adding
  • Reset zone: time to compact or start a fresh conversation — don't keep piling on
  • Danger zone: quality degrades before you hit the wall. Auto-compaction can silently lose detail.
  • Key message: treat the context window like RAM, not a scroll — curate what's in it
